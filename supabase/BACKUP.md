# Резервные копии (бэкапы) FinLit

На бесплатном тарифе Supabase **нет автоматических бэкапов**, поэтому делаем их сами — примерно раз в месяц.
В приложении есть напоминалка: раз в месяц в углу появляется карточка «Пора сделать бэкап» с кнопкой «Готово».

## Зачем
- Случайное удаление данных или ошибочный SQL нечем откатить.
- Бесплатный проект засыпает после 7 дней простоя, а при очень долгом простое (~90 дней) может быть удалён.
- Бэкап — страховка финансовой истории за годы.

## ✅ Способ 1 — экспорт в CSV (рекомендуется, ничего устанавливать не нужно)
1. Открой Supabase -> Table Editor.
2. Для каждой таблицы нажми Export -> Export to CSV:
   `incomes`, `expenses`, `goals`, `goal_contributions`, `currencies`, `months`, `categories`.
3. Сохрани файлы в личный Google Drive / на компьютер.

## Способ 2 — Supabase CLI (ТРЕБУЕТ Docker Desktop)
```
supabase login
supabase link --project-ref ewgrcmswwvbtoxdxkvuv
supabase db dump --data-only -f finlit-backup.sql
```
- ⚠️ `supabase db dump` запускает `pg_dump` внутри Docker-контейнера, поэтому нужен установленный и запущенный Docker Desktop. Без Docker будет ошибка `docker_engine ... cannot find the file`.
- Если Docker ставить не хочешь — используй Способ 1 (CSV) или Способ 3 (pg_dump напрямую).

## Способ 3 — pg_dump напрямую (одна команда, без Docker)
1. Установи клиент PostgreSQL 17 (можно только «Command Line Tools» из установщика EDB). Версия должна совпадать с сервером (PG 17).
2. Возьми строку подключения: Supabase -> Project Settings -> Database -> Connection string (URI), режим Session pooler.
3. Команда (подставь свою строку подключения):
```
pg_dump "CONNECTION_STRING" --data-only --no-owner -f finlit-backup.sql
```
- ⚠️ Пароль от базы в строке подключения — только на твоём компьютере, никому не показывай.

## Восстановление
Примени `schema.sql` + миграции на новый проект, затем залей данные из `finlit-backup.sql` (или импортируй CSV по таблицам).

## ⚠️ Важно
- Репозиторий **публичный** — НИКОГДА не коммить файлы бэкапа (CSV/SQL) в Git: в них твои финансовые данные.
- Храни бэкапы в личном облаке или на компьютере.
- Никогда не публикуй service_role ключ и пароль от базы.
